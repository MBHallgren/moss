If this is your first time installing and using Microbial Outbreak Surveillance System (MOSS), please read and follow the set-up guide carefully.
Additionally, the instructions on how to correctly operate MOSS should be thoroughly studied.

Set up guide:

  1. Initializing your first surveillance system
    2. If you have just downloaded (Cloned) MOSS from github/bitbucket, you will need to either generate a new MOSS system or download an existing one.
    1. Initializing a new MOSS system
      2. Go to the Service tab in the left menu and select "Initialize database". Here to will have to give the following input: 1.) Path to the location in which the database system in to be stored (I.e. "/Guide/usr/dev/MOSSSYSTEM/"), path to where the MOSS software was installed (I.e. "/Guide/usr/programs/MOSS/"), and finally the path to a KMA indexed database which contains reference sequences.
    2. Analysis status
      3. In order to check the status of an analysis, click the results tab in the action bar to the left. If the analysis has finished, it will be shown under "Show Finished Analyses". Otherwise, it will be shown under "Show Running Analyses".
    3. General considerations when using MOSS
      4. Some of the bioinformatical programs run during the MOSS pipeline can be quite computationally requiring, and therefore any user should consider the natural limitations of their hardware. The two primary areas of concern are running multiple analyses in parallel and performing de novo assembly. De novo assembly is extremely demanding in terms of RAM usage, as the De Bruijn graph needs to be stored in the memory in order to determine the next building edge. It is not uncommon for De Bruijn graph of bacterial assemblies of 5M BP size to use upwards of 1-3 Gb RAM. Therefore, if 5 samples are run in parallel and they all fail to identify a proper reference, the RAM usage for graph building could easily be in the 10-20 gb range. For these reasons, it is strongly recommended that a computer build for running MOSS has atleast 16 gb ram - Ideally 32 or even 64.
      Additionally, a good CPU should be utilized, especially if isolate samples are to be run in parallel. The recommended number of cores are atleast 12.

How to use MOSS:

  1. Creating a metadata .csv sheet.
    2. Once a database system has been initialized as described above, the analysis of isolates can begin.
    In order to analyse one or more samples, a metadata sheet must first be created.
    A metadata sheet is a csv-file (comma-separated-values) and can be generated in the Services section of the left menu.
  2. Filling in metadata information.
    3. After the metadata sheet has been created it has to filled in with metadata related to the samples.
    Open the .csv file with an editor of your choosing (excel, etc.) and fill in the data.
    Once the data has been filled in make sure to save the file as a .csv file. Make sure to use the save-as function when saving.
  3. Submit a metadata .csv file for analysis.
    4. Once the metadata has been inserted the .csv file can be submitted for analysis.
    This is done in the Service > Submit metadata sheet for analysis section in the left-side menu.
    Choose either Laptop or Large scale desktop system depending on your operating system. For MacOS multiple-parallel analysis is not available.
    Do not turn off the computer/close the MOSS app during analysis.
  4. Reviewing results.
    5. Once the analysis has been completed, the results can be found in the left-side menu under Results.
