  5. General considerations when using MOSS
    6. Some of the bioinformatical programs run during the MOSS pipeline can be quite computationally requiring, and therefore any user should consider the natural limitations of their hardware. The two primary areas of concern are running multiple analyses in parallel and performing de novo assembly. De novo assembly is extremely demanding in terms of RAM usage, as the De Bruijn graph needs to be stored in the memory in order to determine the next building edge. It is not uncommon for De Bruijn graph of bacterial assemblies of 5M BP size to use upwards of 1-3 Gb RAM. Therefore, if 5 samples are run in parallel and they all fail to identify a proper reference, the RAM usage for graph building could easily be in the 10-20 gb range. For these reasons, it is strongly recommended that a computer build for running MOSS has atleast 16 gb ram - Ideally 32 or even 64.
    Additionally, a good CPU should be utilized, especially if isolate samples are to be run in parallel. The recommended number of cores are atleast 12.
